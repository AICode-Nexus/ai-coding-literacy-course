# 这门课不是教你多背几条 Prompt

它要解决的是一个更接近日常工作的难题：**怎样把人的判断、AI 的能力、真实材料、执行工具和验收机制组织在一起，稳定完成任务。**

<div class="chapter-lead">
  <p>面向具备对话式 AI 基础的校招生。主线从“问答与润色”出发，逐步走到任务契约、上下文工程、Agent 工作系统、质量评测和安全边界。</p>
  <strong>80 + 15′</strong>
</div>

> 学完的标志不是“我听懂了 Agent”，而是你能拿一个真实任务，写清目标、组织材料、让 AI 产出、用证据验收，并知道什么时候必须由人接管。

## 先判断你现在在哪里

| 当前状态 | 常见做法 | 本课中的下一步 |
| --- | --- | --- |
| A · 问答型用户 | 提问、总结、翻译、润色 | 从一句请求升级为任务契约 |
| B · 任务型用户 | 用 AI 完成报告、调研、代码等成果 | 补齐材料、验收与边界 |
| C · 系统型用户 | AI 能搜索、执行、测试并迭代 | 设计 Harness、Evals 和持久上下文 |
| D · 编排型用户 | 使用 Subagent 或 Agent Team | 用依赖与成本判断是否值得并行 |

本课程以 A 类学员为主线，兼顾 B 类学员。已有 Cursor、Claude Code、Codex 等工具基础的学员，可以重点阅读每章的蓝色“进阶层”。

## 你最终要带走的四份成果

1. **一张 AI 协同任务卡**：目标、材料、交付物、验收和边界都能被检查。
2. **一份三层验收清单**：确定性检查、AI 语义评审、责任人确认各司其职。
3. **一套概念边界判断**：不再把 Prompt、Context、MCP、Skill、Agent、Harness 混在一起。
4. **一个 7 天迁移计划**：用一周跑通一条真实工作链，并沉淀可复用资产。

## 内容如何保持不过时

本教材把知识分成三类：

<div class="source-legend">
  <div><small>STABLE</small><h3>稳定方法</h3><p>目标、任务契约、上下文选择、验收、权限与责任边界。作为课程主干。</p></div>
  <div><small>DYNAMIC</small><h3>动态能力</h3><p>具体产品功能、模型名称、窗口大小和工具入口。只讲判断方法，并标注核验日期。</p></div>
  <div><small>INTERNAL</small><h3>内部实践</h3><p>经过提炼的真实案例。保留方法、工件和人工检查点，不传播无法核验的夸张数字。</p></div>
  <div><small>FRONTIER</small><h3>前沿探索</h3><p>多 Agent、长期记忆、活知识库等仍在快速演进的方向。明确写出适用条件与成本。</p></div>
</div>

模型排行榜、未经官方确认的版本参数和“某工具永久最好”等时效内容不进入主线。需要选型时，请查看[工具雷达](/appendix/tool-radar)和[来源与核验记录](/sources)。

## 开始前写下一个真实任务

不要选“学习 AI”这种无法验收的目标。选一个你未来 7 天必须完成的任务，例如：

- 把需求说明预审成待确认问题；
- 为导师整理一份证据清楚的周报；
- 比较三种方案并形成推荐；
- 修改一段代码并交付测试证据；
- 把一次重复流程沉淀为模板或 Skill。

把它保留下来。后面每一章，你都要用同一个任务继续补齐工作系统。
