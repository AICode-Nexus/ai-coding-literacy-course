# VitePress Dual-Mode AI Course Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace the long static workshop page with a VitePress course site that prioritizes an offline-safe 16:9 lecture deck while providing a richer self-study tutorial from the same verified concepts, cases, and sources.

**Architecture:** VitePress owns routing, documentation navigation, search, and static output. Plain ESM data modules under `course/.vitepress/data/` are the single source for agenda, concepts, cases, sources, lecture scenes, presentation navigation, and task-card generation; Vue components consume those modules in both documentation and presentation layouts. A custom theme wraps the default VitePress docs layout and replaces it only for the course home and presentation routes.

**Tech Stack:** VitePress 1.6.4, Vue 3.5.39, JavaScript ESM data modules, Vue SFCs, CSS, Node.js built-in test runner, GitHub Pages, Node 22 CI.

## Global Constraints

- The venue signal is 16:9; design against a logical 1920×1080 stage and scale proportionally.
- The live session is 80 minutes of teaching plus 15 minutes of exchange.
- The primary audience is campus hires with chat-AI basics; advanced Agent/CLI material must be progressive disclosure.
- Lecture mode is primary; every section needs a strong visual, comparison, case, interaction, or demo surface.
- Do not hard-code unverified model rankings, benchmark scores, prices, context-window numbers, or “best model” claims.
- Public factual claims use official first-party sources; internal cases are summarized without sensitive screenshots or data.
- Local development uses `/`; GitHub Pages builds with `/ai-coding-literacy-course/`.
- The production build must work without runtime API calls, remote fonts, or required external images.
- New behavior follows TDD: write the test, observe the expected failure, implement the minimum behavior, and rerun all tests.
- Preserve the dirty `main` checkout by working only in `.worktrees/course-workshop-redesign` on `codex/course-workshop-redesign`.

```js
export function clampScene(index, total) {
  if (!Number.isFinite(index) || total <= 0) return 0;
  return Math.min(Math.max(Math.trunc(index), 0), total - 1);
}

export function sceneFromSearch(search, total) {
  const raw = new URLSearchParams(search).get("scene");
  const oneBased = Number.parseInt(raw || "1", 10);
  return clampScene(Number.isFinite(oneBased) ? oneBased - 1 : 0, total);
}

export function searchWithScene(search, index) {
  const params = new URLSearchParams(search);
  params.set("scene", String(index + 1));
  return `?${params.toString()}`;
}

export function keyToAction(key) {
  return ({
    ArrowRight: "next", ArrowDown: "next", PageDown: "next", " ": "next",
    ArrowLeft: "previous", ArrowUp: "previous", PageUp: "previous",
    Home: "first", End: "last", f: "fullscreen", F: "fullscreen",
    o: "overview", O: "overview",
  })[key] || null;
}
```
